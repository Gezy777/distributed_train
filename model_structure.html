参数量是怎么算出来的？

在流水线并行调优时，可以使用以下公式：

  1. 卷积层 (Conv2d):
    Params=K2×Cin×CoutParams = K^2 \times C_{in} \times C_{out}Params=K2×Cin​×Cout​
    (注：ResNet50 默认 bias=False，所以不加 bias 偏置项)
    例如 conv1: 7×7×3×64=9,4087 \times 7 \times 3 \times 64 = 9,4087×7×3×64=9,408

  2. 批归一化 (BatchNorm2d):
    Params=2×CoutParams = 2 \times C_{out}Params=2×Cout​
    (包含可学习的 γ\gammaγ 和 β\betaβ)

  3. 全连接层 (Linear):
    Params=In×Out+Out(bias)Params = In \times Out + Out(bias)Params=In×Out+Out(bias)

重点解析：参数量是如何计算的？

以 layer4 中的一个 Bottleneck 块（非下采样块）为例：
ResNet50 的 Bottleneck 包含三层卷积：1×11\times11×1, 3×33\times33×3, 1×11\times11×1。

  1. 第一层 (1×11\times11×1 Conv)：
    • 输入 2048，输出 512。
    • Params=1×1×2048×512=1,048,576Params = 1 \times 1 \times 2048 \times 512 = 1,048,576Params=1×1×2048×512=1,048,576
  2. 第二层 (3×33\times33×3 Conv)：
    • 输入 512，输出 512。
    • Params=3×3×512×512=2,359,296Params = 3 \times 3 \times 512 \times 512 = 2,359,296Params=3×3×512×512=2,359,296
  3. 第三层 (1×11\times11×1 Conv)：
    • 输入 512，输出 2048。
    • Params=1×1×512×2048=1,048,576Params = 1 \times 1 \times 512 \times 2048 = 1,048,576Params=1×1×512×2048=1,048,576

结论：可以看到，越往后的层（Layer3, Layer4），由于通道数（Channels）翻倍，参数量呈指数级爆炸。Layer4 只有 3 个块，但参数量却是 Layer1 的 70 倍左右。

流水线并行的“潜规则”

  • 前 3 个阶段 (Stem + Layer 1 + Layer 2)：虽然层数非常多（包含 7 个 Bottlenecks），但总参数量加起来才不到 150万。
  • 最后 2 个阶段 (Layer 3 + Layer 4)：参数量高达 2200万，占据了全模型的 86% 以上。

这意味着：
如果你使用 partition_method='parameters' 进行 2 卡流水线并行：

  • GPU 0：会分到 conv1 一直延伸到 layer3 的绝大部分。
  • GPU 1：可能只分到 layer4 和 fc。

虽然这样**显存（Memory）**是均衡了，但 **GPU 0 的计算量（FLOPs）**会大得多，因为它处理的特征图尺寸（112×112112\times112112×112, 56×5656\times5656×56）远大于 GPU 1（7×77\times77×7）。

要深入理解 ResNet50 的每一层，我们需要追踪两个并行发生的变化：特征图（Feature Map）的空间收缩与参数量（Parameters）的通道膨胀。

  1. 输出尺寸计算 (Output Shape):

4. 关键结论：参数与格式的“背离”

在你的流水线并行（PP）任务中，你会发现一个有趣的现象：

  • 数据吞吐量 (Data Bandwidth)：
    在前几层（Stem, Layer 1），特征图尺寸巨大（如 112×112112\times112112×112），每层输出的 Tensor 体积很大，跨卡传输非常耗时。
  • 计算/显存压力 (Compute/Memory)：
    在后几层（Layer 3, Layer 4），卷积核通道数翻倍，参数量呈几何倍数增加，显存占用极高，但特征图缩小到了 7×77\times77×7，传输反而很快。

总结规律：

  • 输入输出格式：主要由 Stride 决定，负责“瘦身”。
  • 参数量：主要由 Channels 的乘积决定，负责“增重”。

在卷积神经网络（CNN）中，通道数（Channels） 是理解特征提取的核心维度。你可以从**“深度”、“特征库”和“组合逻辑”**三个层面来直观理解：

1. 通道的直观类比：特征的“提取维度”

  • 输入层（RGB）： 想象一张照片。它有三个通道（红、绿、蓝）。每一个通道都是一张灰度图，记录了该颜色在不同位置的强度。
  • 隐藏层（Feature Maps）： 随着网络变深，通道数从 3 变成 64、256 甚至 2048。这时候，每一个通道不再代表颜色，而是代表一种**“视觉特征”**。
    • 第 1 个通道可能专门负责找“横向边缘”；
    • 第 2 个通道专门负责找“圆弧”；
    • 第 100 个通道可能在找“猫的耳朵形状”。

结论： 通道数越多，意味着模型在同一位置能同时观察到的特征种类越丰富。

2. 参数量计算：通道是如何相互作用的？

理解通道最难的一点是：卷积核并不是在一个通道上滑动，而是在所有输入通道上同时“深挖”。

假设输入有 Cin=3C_{in} = 3Cin​=3 个通道，输出要得到 Cout=64C_{out} = 64Cout​=64 个通道：

  1. 一个卷积核（Filter）的构成： 为了处理 3 个输入通道，这一个卷积核本身必须有 3 层（即深度为 3）。
  2. 一次卷积的过程： 这 3 层卷积核分别与 3 个输入通道做乘加运算，最后求和，塌陷成一个值。这一个值就是输出特征图中一个通道的一个像素点。
  3. 为什么参数会爆炸： 如果你要输出 64 个通道，你就需要 64 个这样“有深度的卷积核”。

计算公式：
Params=K×K×Cin×CoutParams = K \times K \times C_{in} \times C_{out}Params=K×K×Cin​×Cout​

  • K×KK \times KK×K: 卷积核的窗口大小（如 3×33 \times 33×3）。
  • CinC_{in}Cin​: 你有多少个特征来源。
  • CoutC_{out}Cout​: 你想提取多少种新特征。

3. ResNet50 中的通道演变逻辑

在 ResNet50 的 Bottleneck 结构中，通道数的变化非常有节奏，它遵循 “压缩 →\to→ 提取 →\to→ 扩张” 的逻辑：

  1. 1x1 卷积（压缩通道）： 将通道数从 1024 砍到 256。这是为了减少计算量，把主要的特征“浓缩”一下。
  2. 3x3 卷积（特征提取）： 在浓缩后的 256 个通道上进行复杂的空间特征提取。
  3. 1x1 卷积（扩张通道）： 将通道数从 256 恢复到 1024（甚至是 2048）。这是为了把提取到的特征重新映射到高维空间，方便下一层进行更复杂的组合。

4. 总结：通道 vs 分辨率

你可以把卷积网络看作一个不断改变数据形状的工厂：

  • 分辨率（H, W）降低： 意味着模型在“宏观”上看图像，视野越来越广（从像素到物体局部到整个物体）。
  • 通道数（C）增加： 意味着模型在“微观”上分析语义，理解越来越深（从线条到纹理到复杂的语义概念）。

在你的双卡流水线并行中：

  • GPU 0 处理的是“大分辨率、少通道”的数据，计算密集。
  • GPU 1 处理的是“小分辨率、多通道”的数据，参数密集（显存占用大）。
